## Available Widgets

### Data Visualization

| Widget       | Purpose                                | Best For                           |
| ------------ | -------------------------------------- | ---------------------------------- |
| **Chart**    | Line, bar, area, or pie visualizations | Trends, comparisons, distributions |
| **Progress** | Goal completion percentage             | Targets with defined endpoints     |
| **Counter**  | Running total display                  | Cumulative metrics                 |

### Activity Tracking

| Widget        | Purpose                  | Best For                  |
| ------------- | ------------------------ | ------------------------- |
| **Streak**    | Consecutive day tracking | Habit monitoring          |
| **Timer**     | Countdown display        | Deadlines, time-boxing    |
| **Stopwatch** | Elapsed time tracking    | Active session monitoring |
| **Rating**    | Star-based averages      | Quality metrics           |

<Tip>
  Dashboards work best when widgets pull data from other helpers. Create your trackers and logs first, then build a dashboard to visualize them.
</Tip>

## Dashboard vs. Other Templates

<Info>
  **Dashboard** = Visualization and monitoring, minimal data entry\
  **Tracker** = Data capture with built-in stats\
  **Form** = Pure data entry, no visualization
</Info>

| Feature         | Dashboard      | Tracker         | Form             |
| --------------- | -------------- | --------------- | ---------------- |
| Primary purpose | View data      | Capture + view  | Capture only     |
| Widgets         | Multiple       | Limited         | None             |
| Data entry      | Optional       | Primary         | Primary          |
| Charts          | Multiple types | Single optional | None             |
| Best for        | Monitoring     | Daily logging   | One-time capture |

## Building Effective Dashboards

### Layout Principles

<Steps>
  <Step title="Lead with key metrics">
    Place your most important numbers at the top. Users should see critical data without scrolling.
  </Step>
  <Step title="Group related widgets">
    Keep related data togetherâ€”financial metrics in one area, health metrics in another.
  </Step>
  <Step title="Use charts sparingly">
    One or two well-chosen charts communicate more than five competing visualizations.
  </Step>
  <Step title="Include context">
    Show trends and comparisons, not just current values. "500" means nothing without "up 20% from last week."
  </Step>
</Steps>

## Configuration

### In Edit Mode

Access via the **Edit** button to:

| Action            | Description                          |
| ----------------- | ------------------------------------ |
| **Add widget**    | Choose from available widget types   |
| **Remove widget** | Delete widgets you don't need        |
| **Configure**     | Set data sources and display options |
| **Reorder**       | Drag widgets to rearrange            |
| **Theme**         | Change colors and icon               |

### Connecting Data

Widgets can pull data from:

- **Same helper** â€” Fields within the dashboard
- **Other helpers** â€” Cross-reference your trackers and logs
- **Calculated values** â€” Derived from formulas

<Note>
  Cross-helper data requires helpers to share the same owner. You can't visualize other users' data on your dashboard.
</Note>

## Tips & Best Practices

<AccordionGroup>
  <Accordion title="Start with questions" icon="circle-question">
    Before building, ask: "What questions should this dashboard answer?" Design widgets to answer those specific questions.
  </Accordion>
  <Accordion title="Less is more" icon="minimize">
    A dashboard with 3-4 well-chosen widgets beats one with 10 competing for attention. Prioritize ruthlessly.
  </Accordion>
  <Accordion title="Update frequency matters" icon="clock">
    Don't create dashboards for data that rarely changes. Dashboards shine when data updates frequently and trends emerge.
  </Accordion>
  <Accordion title="Mobile-first design" icon="mobile">
    Test your dashboard on mobile. Widgets stack vertically on small screensâ€”ensure the order makes sense.
  </Accordion>
</AccordionGroup>
