## List vs. Other Templates

<Tip>
**List** = One-time tasks, manually managed, persistent  
**Checklist** = Same tasks daily, auto-reset  
**Tracker** = Recurring data logging with trends
</Tip>

| Feature | List | Checklist | Tracker |
|---------|------|-----------|---------|
| Items persist | ‚úÖ Until deleted | ‚ùå Reset daily | ‚úÖ Historical log |
| Custom fields | ‚úÖ Per item | ‚ùå Items only | ‚úÖ Per entry |
| Completion tracking | ‚úÖ Manual | ‚úÖ Daily | ‚ùå No checkbox |
| Time-based reset | ‚ùå Never | ‚úÖ Daily | ‚ùå Never |
| Best for | Tasks, collections | Routines | Data logging |

## Custom Fields

Lists support rich data capture for each item:

### Supported Field Types

| Type | Example Use |
|------|-------------|
| **Text** | Item name, notes, description |
| **Number** | Quantity, priority, price |
| **Select** | Category, status, priority level |
| **Checkbox** | Urgent flag, requires follow-up |
| **Date** | Due date, purchase date |
| **Image** | Product photo, reference image |
| **Calculated** | Total cost (quantity √ó price) |

### Example: Shopping List Fields

Item (text, required)
Quantity (number, default: 1)
Category (select: Produce, Dairy, Meat, Pantry, Frozen)
Price estimate (number, optional)
Total (calculated: quantity √ó price)

## Managing Items

### Completion

Toggle items complete by:
- **Tapping checkbox** ‚Äî Marks done with strikethrough
- **Voice command** ‚Äî "Mark dentist complete"
- **Telegram** ‚Äî "done: dry cleaning"

<Info>
Completed items remain visible (with strikethrough styling) until you delete them. This lets you review what you've accomplished.
</Info>

### Editing

Tap any item to:
- View full details
- Edit any field
- Add notes or attachments
- Delete the item

### Bulk Operations

From the list view:
- **Delete completed** ‚Äî Remove all done items at once
- **Export** ‚Äî Download list as CSV/PDF

## Configuration

### In Edit Mode

Access via the **Edit** button to:

| Setting | Options |
|---------|---------|
| **Add fields** | Create new data fields |
| **Remove fields** | Delete unnecessary fields |
| **Reorder fields** | Change display order |
| **Set required** | Mark fields as mandatory |
| **Default values** | Pre-fill common values |
| **Theme** | Change colors and icon |

### Field Options

Each field supports:

| Option | Description |
|--------|-------------|
| **Required** | Must be filled to save item |
| **Placeholder** | Hint text in empty fields |
| **Default** | Pre-filled value |
| **Options** | For select fields, the choices |

### Widgets

Add optional widgets:

| Widget | Purpose |
|--------|---------|
| **Counter** | Total items or sum of a field |
| **Progress** | Completion percentage |
| **Chart** | Items by category or over time |

## Tips & Best Practices

<AccordionGroup>
  <Accordion title="Use categories wisely" icon="folder">
    Add a select field for categories (e.g., Produce, Dairy) to group similar items. This makes long lists easier to scan.
  </Accordion>
  
  <Accordion title="Set sensible defaults" icon="sliders">
    If most items share a value (quantity: 1, priority: normal), set it as the default to speed up entry.
  </Accordion>
  
  <Accordion title="Keep primary field short" icon="text">
    The item name should be scannable at a glance. Use notes or description fields for longer details.
  </Accordion>
  
  <Accordion title="Regular cleanup" icon="broom">
    Periodically delete completed items to keep your list focused. Use "Delete completed" for bulk cleanup.
  </Accordion>
</AccordionGroup>
